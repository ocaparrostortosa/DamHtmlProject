Introducción a JSON

¿Qué es JSON?

Su acrónimo significa JavaScript Object Notation, es un formato de texto ligero para el intercambio de datos JS, nos va a asegurar que nuestro código, nuestro programa JS corra correctamente en cualquier navegador basado de renderizado HTML5 u otro sistema operativo basado en HTML5.

JSON es una manera de intercambiar datos de manera mucho más eficciente, rápida y óptima, pues podemos intercambiar el mismo objeto con el que estamos trabajando, sin recesidad de serializar o hacer marshalling de datos

¿Qué es serializar?

De forma concisa es pasar un Objeto a un array de bytes y viceversa. Sirve para poder trasmitir y escribir objetos a medios externos como webservices, archivos para más información click aquí

¿Qué es Marshalling?

Marshalling: Significa crear un documento, por ejemplo, XML de un árbol de contenido(En nuestro caso el arbol del contenido seria un Objeto Java).

Estructuras JSON

Objetos de datos (Clave:Valor)

Aunque primero debemos definir que es un objeto. En java un objeto es una unidad dentro de un programa que consta de un estado y comportamiento.
de manera respectiva de datos almacenados y la tarea que debe realizar.

Un ejemplo de Clave:Valor es:

{clave1:valor1, clave2:valor2}


Ejemplo

{ "nombre" : "Javi",
"apellidos" : "Collado",
"e-mail" : javicb55@gmail.com }
tambien se puede formar con este formato:

persona.nombre = "Juan";
persona.apellidos = "Gutierrez";
persona.email = "jgutierrez@iesvirgendelcarmen";

Estructura Array

Los Arrays son listas de objetos de datos tipo (Valor:Clave) como hemos visto anteriormente. También se caracterizan por llevar corchetes

Un ejemplo

["plátanos", "manzanas", "peras"]

Donde tenemos un vector con 3 elementos. El elemento "0" son los plátanos, el elemento "1" las manzanas y el elemento "2" peras.

También los arrays pueden ser los valores claves de nuestro objeto, podemos definir una clave "Alumnos" que sea un vector con los alumnos de nuestro instituto

"Alumnos": [
{"PrimerNombre": "Juana", "Apellido": "Arco"},
{"PrimerNombre": "William", "Apellido" : "Walace"},
{"PrimerNombre": "Barba", "Apellido": "Roja"}
]

Una manera más práctico sería creando 2 archivos JSON para intercambiar datos, en este ejemplo intercambio los tutoriales del grupo 1 con los del grupo 3 con solo cambiar el archivo .json

JSON SCHEMA

JSON-Schema Es un formato JSON para describir datos en JSON. Lo que ofrece un contrato para definir los datos requeridos para una aplicación dada y la forma de interactuar con él

Ejemplo

{ "$schema" : "http://json-schema.org/draft-04/schema#“,
"título" : "Producto",
"descripción": "Un producto de nuestro catálogo"
"tipo" : "object",
"propiedades":{
"id":{
"descripción" : "Forma de identificar el producto",
"tipo" : "integer" (Número Sin Decimal)
},
nombre:{
"Descripción" : "Nombre del producto",
"type" : "string" (Cadena)
},
precio: {
"type": "Number"(Número)
"mínimo": 0,
"exclusiveMinimum" : true
}
},
"required"(Requiere): ["id","nombre","precio"]
Que validaría como válidos JSONs como este:
{
"id" : 1,
"nombre" : "Mesa madera",
"precio" : "5.5"
} }
